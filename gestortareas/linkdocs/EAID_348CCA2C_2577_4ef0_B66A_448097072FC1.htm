NOMBRE OPERACIÓN
generarInformesEstadoTableros
VERSIÓN
1.0.0
TRANSACCIONAL
NO
AUDITADO
NO
PERIODICIDAD
Diario, Lunes a Viernes, a las 06:00.
DESCRIPCION
 - El sistema obtiene la relación de los participantes que son autoridades informadas de tableros, excluyendo los archivados.
- Solo se considerarán las organizaciones  que tienen habilitado el envío de informes (atributo "generacionInformes" a TRUE)

- Para cada participante:
      - Se obtienen los usuarios afectados: Para Miembros, el usuario asociado, y para Grupos, los usuarios de tipo AUTORIDAD.
      - Se descartan los usuarios que no tienen activada la preferencia "informesEstadoTableros".
      - Si queda al menos un usuario tras el punto anterior, se genera un informe en PDF de tipo "Informe de Estado de tableros", mediante la API "InformesAPI.informeEstadoTableros", pasando el parametro del Participante.
      - Se genera y remite a cada usuario un mensaje de correo electrónico con la plantilla "Informe de Estado de tableros", con el informe anexado.

 
RESTRICCIONES
PRECONDICIONES
 - El job solo se ejecutará si la propiedad del sistema "EJECUTAR_JOB_INFORMES" tiene valor "1".

DESCRIPCION

      - Se genera y remite a cada usuario un mensaje de correo electrónico con la plantilla "Informe Resumen de Grupo", con el informe anexado.
