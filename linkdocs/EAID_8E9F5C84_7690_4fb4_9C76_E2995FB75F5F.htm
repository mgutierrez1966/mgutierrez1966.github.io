LeerNivelArbolApoyoOacise
URL
TRANSACCIONAL
NO
AUDITADO
SI
DESCRIPCIÓN
 Devuelve un nivel del árbol de nodos a cargo apoyados por un determinado CECOM / OACISE, a partir de un nodo superior especificado.
 
DATOS ENTRADA
NOMBRE
TIPO
REQUERIDO
DESCRIPCION

SI
ID del CECOM / OACISE que realiza el apoyo.
nodoSuperior
ID
NO
ID del NodoOrganicaEt a partir del cual se buscan los nodos apoyados por el CECOM especificado. Si es NULL, se busca desde la raiz de la orgánica del ET.
DATOS SALIDA
NOMBRE
TIPO
DESCRIPCION
nodosApoyados
Lista
Lista de nodos de la orgánica que son apoyados por el CECOM especificado, en el nivel raiz, o bien en el siguiente nivel bajo el nodo superior especificado.
PROCESO
- Se realiza una consulta de la entidad NodoCargoRecurso, filtrada por aquellas cuyo "oacise" es el Cecom especificado, y ordenado por el atributo "path". Si el parámetro "nodoSuperior" tiene valor, además se filtra para que la lista solo incluya los nodos subordinados al especificado.
- La lista de obtenida se filtra para incluir solamente un nivel del árbol. Para ello se procesa cada elemento (en el orden obtenido), identificando los nodos subordinados que forman parte del siguiente nivel:
     - Si el nodo procesado es el especificado como "nodoSuperior", se ignora
     - El primer nodo diferente del "nodoSuperior" se almacena como resultado, y su path se guarda como referencia;
     - Para los siguientes nodos:
        - Si el path del nodo procesado incluye el path de referencia, se ignora;
      - Si el path del nodo procesado no incluye el path de referencia, se almacena como resultado, y su path se guarda como nuevo path de referencia;

- Finalmente el servicio devuelve los nodos almacenados en la lista de resultados .

AgregarNodoCargoApoyadoPorOacise
URL
TRANSACCIONAL
SI
AUDITADO
SI
DESCRIPCIÓN
Establece el CECOM/OACISE de un nodo a cargo de recursos y de todos sus nodos subordinados. También modifica las solicitudes de mantenimiento de los recursos de dichos nodos, y que estén pendientes de tramitar por el CECOM, para que se asignen al nuevo CECOM.
DATOS ENTRADA
NOMBRE
TIPO
REQUERIDO
DESCRIPCION
cecom
ID
SI
ID del CECOM / OACISE que realiza el apoyo.
nodo
ID
SI
ID del nodoCargoRecursos a modificar.
DATOS SALIDA
NOMBRE
TIPO
DESCRIPCION
PROCESO
- Se comprueba que el ID corresponde a un CECOM
- Se obtiene el nodoCargoRecursos correspondiente al ID, y se actualiza su atributo "oacise" con el Cecom pasado como parámetro.
- Se ejecuta una actualización en lote de todos los nodosCargoRecurso subordinados, estableciendo el "oacise" al Cecom pasado como parámetro.
- Las solicitudes de mantenimiento con "nodoCispoc" igual al nodo agregado o a cualquiera de los nodos subordinados, y que se encuentren en un estado de tramitación "NUEVA", deben ser reasignadas al nuevo CECOM (atributo "cecomOacise").
