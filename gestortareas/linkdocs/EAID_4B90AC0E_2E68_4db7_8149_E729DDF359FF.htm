 Permite buscar los datos de un usuario de DICODEF por su UID (nombre de usuario).
El botón "Comprobar" solo está habilitado si el campo "UID" no está vacío.

 Muestra parte de los datos devueltos por la llamada al servicio de búsqueda de usuarios de DICODEF.

 Habilitado solo si se ha localizado un usuario DICODEF que no esté agregado ya a la organización.

 DESCRIPCION DEL PROCESO A EJECUTAR
Clic
Comprobar
 Realiza 2 acciones:
  • Primero comprueba si el usuario ya está dado de alta en la organización, intentando recuperar los datos a través del servicio REST de la entidad Usuario, filtrando por UID. Si el usuario ya está dado de alta en la organización, se informa de ello y se muestran en los campos los datos del usuario asociado.
  • Si el usuario no está dado de alta en la organización, llama al servicio API leerUsuarioDicodef con el parámetro del UID. Si el servicio se ejecuta correctamente, se muestran los datos en el bloque "Datos de DICODEF". Si el servicio responde con un error, se muestra al usuario.
 
Clic
Dar acceso al usuario
 Se llama a la API AdminAPI/altaUsuarioOrganizacion, pasando el UID del usuario y el ID de la organizacion.

Si la llamada es correcta se cierra el formulario y se devuelve como datos el usuario agregado.
