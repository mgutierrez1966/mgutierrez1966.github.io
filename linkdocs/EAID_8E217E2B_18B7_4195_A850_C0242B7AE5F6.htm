 Formulario de creación o modificación de una solicitud de mantenimiento adaptativo para uno o varios recursos HW.
Para modificación, solo permite modificar los datos básicos, pero no la lista de recursos asociados.

  • solicitudMantAdaptaHw: Opcional. Objeto a modificar (en caso de modificación)

Datos generales
Tab panel
Pestaña que contiene los campos con los datos generales de la solicitud.
Formulario
Formulario para rellenar los datos generales de la solicitud.
Campo "Código"
Texto
Muestra el código de la solicitud, o "Pendiente de asignar" para nuevas solicitudes.
Campo "Motivo de la solicitud"
Se corresponde con el atributo "Descripción" de la solicitud de mantenimiento.
Campo "Documento autorización"
Selector de fichero
 Permite seleccionar un documento que autoriza el mantenimiento adaptativo. Admite formato PDF, con un tamaño máximo de 10 MB.
Dispone de un botón para seleccionar un fichero local (habilitado si no se ha especificado ningún fichero), y otro botón para eliminar el fichero previamente cargado (habilitado si ya se ha especificado un fichero).
En caso de estar cargado un fichero, se muestra un enlace con el nombre del mismo y su extensión. Si no hay fichero asociado se muestra el texto "(Seleccione un fichero)"
Campo "Criticidad"
Muestra datos de la entidad "TipoCriticidadHw" (servicio REST)
Campos "CISPOC o contacto" y "Teléfono"
Se inicializan con el nombre completo y el teléfono del usuario CISPOC o DIRECTOR que crea la solicitud.
Campos "Dirección", "CP" y "Provincia"
- Para rol CISPOC: Se inicializan con los datos equivalentes de la UCO apoyada por el usuario.
- Para rol DIRECTOR: Los datos se inicializan en blanco
Campo "Almacén de baja"
Selector de UCO
 Habilitado solo para el rol DIRECTOR. Permite seleccionar un nodo tipo ALMACEN de la estructura orgánica (atributo "almacenBaja").
En caso de estar seleccionado un almacén , se muestra el nombre largo del mismo. Si no hay almacén seleccionado se muestra el texto "(Ningún almacén seleccionado)"
Campo "Comentarios Director"
** INICIO ARIGIS-1136
Solo es modificable por el rol DIRECTOR. Para el resto de roles aparece como solo lectura.
** FIN ARIGIS-1136

Visible solamente si la vista se muestra para CREACIÓN (parámetro de entrada 'solicitudMantAdaptaHw' vacío).

- Asignar peso: Habilitado si hay algún recurso seleccionado en la lista
** FIN ARIGIS-1083
Tabla de recursos
Grid
 Grid con la lista de recursos asociados a la solicitud de mantenimiento adaptativo.
Toma los datos de un almacén local con la estructura de la entidad RecursoHw, inicialmente vacío.
Inicialmente ordenada por NI ascendente.
** INICIO ARIGIS-1083
En la cabecera de la tabla se muestra un texto con el número de recursos de la lista, y el peso total de los mismos.
** FIN ARIGIS-1083

 DESCRIPCION DEL PROCESO A EJECUTAR
Inicializar vista
Para creación:
- Se llama a la API MantenimientoHwAPI.obtenerContratoMantenimientoHwEnVigor(), para obtener el objeto ContratoMantenimiento en vigor actualmente. Si la llamada devuelve un objeto nulo, se muestra un error al usuario informando de que "No se puede crear la solicitud porque no existe ningún contrato de mantenimiento de HW en vigor actualmente".
- Crear un objeto SolicitudMantAdaptaHw nuevo e inicializar valores por defecto:
      - criticidad: BAJA
      - datos de CISPOC y contacto: ver comentarios a los componentes.

Para edición:
- Tomar valores del objeto a editar
- Iniciar edición del modelo
- Recuperar la lista de recursos asociados a la solicitud.
Clic
Añadir recursos por consulta
 Muestra la vista "FormSeleccionRecursosHw", con los atributos:
- admiteSeleccionMultiple: true
- filtroPorEstado: ALTA

Si la vista tiene éxito, se añaden los recursos seleccionados a la lista de los recursos de la solicitud de mantenimiento, con las siguientes restricciones:
- Todos los recursos de la lista deben de pertenecer a la misma categoría  hardware. Si no es así se mostrará un error al usuario y no se añadirán los recursos.
- Todos los recursos deben estar a cargo de la misma UCO. Si no es así se mostrará un error al usuario y no se añadirán los recursos
- Un mismo recurso no puede estar aparecer más de una vez en la lista. En este caso no se mostrará error, pero los recursos duplicados no se añadirán.
 
Clic
Añadir recursos por lista de NI/NS
 Muestra la vista "FormSeleccionarRecursosHwPorLista".
Si la vista tiene éxito, se obtiene la información de los recursos mediante el servicio REST de RecursoHw, filtrando por los NI y NS especificados.
Si el servicio REST no devuelve ningún resultado, se informa de ello al usuario y se aborta la operación.
Si el servicio REST devuelve al menos un recurso, se añaden los recursos seleccionados a la lista de los recursos de la solicitud de mantenimiento, con las mismas restricciones que las indicadas para "Añadir recursos por consulta"
 
Clic
Quitar recursos
 Solicita confirmación para quitar de la reasignación los recursos seleccionados.
Elimina de la lista los recursos seleccionados.
 
Clic
Asignar peso
 ** INICIO ARIGIS-1083
Muestra la vista FormAsignarPesoRecursos, pasando la lista de recursos seleccionados.
Si la vista tienen éxito, se actualiza la información de la lista para reflejar el peso de los recursos, y se actualiza la cabecera de la tabla con el número de recursos y el peso total de los mismos.
** FIN ARIGIS-1083
Clic
Seleccionar documento justificativo
Muestra el explorador de ficheros del sistema para seleccionar un documento
Cambio de valor
Documento justificativo
Si el fichero no tiene formato PDF o supera los 10 MB, se mostrará un error informando de dichas limitaciones.
Clic
Eliminar documento justificativo
Se llama al servicio REST de la entidad Documento, con la operación DELETE. Se elimina la asociación con el documento y se muestra "(Ningún almacén seleccionado)".
Clic
Guardar
 ** INICIO ARIGIS-1083
Si alguno de los recursos de la lista no tiene asignado un peso, se muestra el error "Todos los recursos asociados a la solicitud deben tener un peso asignado"
** FIN ARIGIS-1083

Para edición:
- Finalizar edición del modelo
- Guardar el modelo (servicio REST de SolicitudMantAdaptaHw)

Para creación:
- Si la lista de recursos asociados está vacía, se muestra el error "Especifique al menos un recurso a incluir en la solicitud de mantenimiento", y se cancela la operación.
- Llamar a la API MantenimientoHwAPI.CrearSolicitudAdaptativo(), pasando el nuevo objeto SolicitudMantenimientoAdaptativo y la lista de los identificadores de los recursos asociados a la nueva solicitud.
- Si la operación es correcta, se muestra un mensaje al usuario informando de que la solicitud se ha creado correctamente con el número XXXXXXX (devuelto por el servicio API).

En ambos casos, se guarda el objeto creado o modificado en la configuración de salida.
