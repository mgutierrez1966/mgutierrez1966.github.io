Formulario
Formulario para rellenar los datos de la solicitud. Modal.
Campo "Codigo"
Texto
Muestra el código de la solicitud, o "Pendiente de asignar" para nuevas solicitudes.
Campo "Recurso averiado"
Combobox
 Permite seleccionar el recurso averiado introduciendo el NI o número de serie, o bien  a traves de una  vista de seleccion de recursos. Muestra el recurso seleccionado para la solicitud.
Si no está vacío, se muestran los siguientes datos: NI - Descripción (Número de serie).
El cuadro de texto obtiene sus valores del servicio REST de la entidad RecursoHw.
No se muestra el trigger del desplegable. Debe forzar a que el valor esté en la lista.
Campo "Requiere equipo de sustitución..."
Checkbox
 Por defecto "false".
Visible solamente si hay un recurso seleccionado en "recurso averiado", y si la categoría a la que pertenece el articulo asociado al recurso tiene el campo "permiteEquipoSustitucion" a "true".
 
Campo "Tipo de avería"
Muestra datos de la entidad "TipoAveria" (servicio REST)
Campo "Criticidad"
Muestra datos de la entidad "TipoCriticidadHw" (servicio REST)
Campo "Responsable recurso"
Se inicializa con el dato "responsable" del recursoHw seleccionado.
Campos "CISPOC o contacto" y "Teléfono"
Se inicializan con el nombre completo y el teléfono del usuario CISPOC que crea la solicitud.
Campos "Dirección", "CP" y "Provincia"
Se inicializan con los datos equivalentes del nodo donde se encuentra ubicado el recurso averiado.
 
Campo "Comentarios Director"
Solo es modificable por el rol DIRECTOR. para el resto de roles aparece como solo lectura.
 
Barra de botones
Toolbar

 DESCRIPCION DEL PROCESO A EJECUTAR
Inicializar vista
Para creación:

- Se llama a la API MantenimientoHwAPI.obtenerContratoMantenimientoHwEnVigor(), para obtener el objeto ContratoMantenimiento en vigor actualmente. Si la llamada devuelve un objeto nulo, se muestra un error al usuario informando de que "No se puede crear la solicitud porque no existe ningún contrato de mantenimiento de HW en vigor actualmente".

- Crear un objeto SolicitudMantenimientoCorrec nuevo e inicializar valores por defecto:
      - recursoAveriado: Valor del parámetro "recurso" si viene informado, de lo contrario a nulo. Se deberá inicializar la información del recurso mediante su servicio REST.
      - criticidad: BAJA    

Para edición:
- Tomar valores del objeto a editar
- Iniciar edición del modelo
 
Cambio de valor
Recurso (NI)
 Se actualiza el atributo "recursoAveriado" del nuevo objeto, y se refresca la información mostrada en el texto.

El campo "Requiere equipo de sustitución..." se actualiza en función del nuevo recurso seleccionado:
    - Si la categoría a la que pertenece el articulo asociado al recurso tiene el campo "permiteEquipoSustitucion" a "true", el campo se hace visible.
    - En caso contrario, el campo se oculta y se actualiza a "false"
 
Clic
Seleccionar recurso
 ** INICIO ARIGIS-1276
Se muestra la vista FormSeleccionRecursosHw, con la siguiente configuración:
      - familia: HW
      - admiteSeleccionMultiple: falso
      - filtroPorEstado: ["ALTA"]
      - nodoRaizFiltroOrganica: ID de la DependenciaEt apoyada por el CISPOC que realiza la solicitud
** FIN ARIGIS-1276

Si la vista se cierra con éxito, se actualiza el atributo "recursoAveriado" del nuevo objeto, y se refresca la información mostrada en el texto.

Guardar

En ambos casos, se guarda el objeto creado o modificado en la configuración de salida.
