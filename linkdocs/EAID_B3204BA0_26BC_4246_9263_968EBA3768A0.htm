OBSERVACIONES
Componente que implementa la interfaz ProcesadorEmailDelegate, y que procesa los correos remitidos por la empresa en relación a las solicitudes de mantenimiento.


OPERACIONES

NOMBRE
ProcesarMensaje
DESCRIPCIÓN
 Procesa un mensaje recibido de la empresa. Valida que el cuerpo del mensaje es un XML válido según la estructura de los mensajes XML definidos en la interfaz con la empresa.
Deserializa el contenido del XML en objetos de negocio, y en función del tipo de mensaje, llama a diferentes operaciones del servicio MantenimientoHwService que realizan la tramitación de la solicitud.
** INICIO ARIGIS-451
Valida que el remitente de correo es el mismo que el valor del atributo "contratoMantenimiento.emailIntercambio" de la solicitud a la que corresponde el mensaje (CODGISMI).
** FIN ARIGIS-451

PROCESO
  • Se identifica el tipo de mensaje según el campo "TIPO" del XML
  • Si el tipo del mensaje es "CONFIRMACION RECEPCION", se validará el XML según el formato XML-RECEPCION-PETICION-MANTENIMIENTO, y se llamará a la operación MantenimientoHwService.ProcesarRecepcionEmpresa
  • Si el tipo del mensaje es "LISTA ARTICULOS", se validará el XML según el formato XML-LISTA-ARTICULOS, y se llamará a la operación MantenimientoHwService.ProcesarListaArticulosEmpresa
  • Si el tipo del mensaje es "CIERRE TECNICO", se validará el XML según el formato XML-CIERRE-PETICION-MANTENIMIENTO, y se llamará a la operación MantenimientoHwService.ProcesarCierrePeticionEmpresa
  • Si el tipo del mensaje es "CONFIRMACION RECEPCION PARTE QUEJA", se validará el XML según el formato XML-RECEPCION-PARTE-QUEJA, y se llamará a la operación MantenimientoHwService.ProcesarRecepcionParteQuejaEmpresa
  • Si el tipo del mensaje es "RESOLUCION PARTE QUEJA", se validará el XML según el formato XML-RESOLUCION-PARTE-QUEJA, y se llamará a la operación MantenimientoHwService.ProcesarResolucionParteQuejaEmpresa
  • Finalmente, se crea un objeto de tipo MensajeEmpresaMantenimiento, con los datos de la solicitud de mantenimiento asociada, del mensaje y de su procesado. En caso de que se hayan producido errores de validación del XML, o bien alguna excepción de negocio en el servicio MantenimientoHwService, los mensajes de error producidos se guardarán en el nuevo objeto.
 
DEPENDENCIAS
RESTRICCIONES
PRECONDICIONES
 - El mensaje debe ser un fichero de texto XML valido según alguno de los esquemas definidos para la interfaz con la empresa de mantenimiento. Esto incluye, la estructura, datos requeridos, formato de datos, etc.
** INICIO ARIGIS-451
Valida que el remitente de correo es el mismo que el valor del atributo "contratoMantenimiento.emailIntercambio" de la solicitud a la que corresponde el mensaje (CODGISMI).
** FIN ARIGIS-451
- Se aplicarán las restricciones definidas para cada uno de los campos del mensaje.
 
POSTCONDICIONES
- Se ha creado un objeto MensajeEmpresaMantenimiento asociado a la solicitud de mantenimiento, con los datos del mensaje, errores, etc.
INVARIANTES
 - El mensaje XML, una vez validado, será transformado en uno o varios objetos de negocio, que serán enviados como parámetros al correspondiente servicio de negocio para la tramitación de la solicitud de mantenimiento.
