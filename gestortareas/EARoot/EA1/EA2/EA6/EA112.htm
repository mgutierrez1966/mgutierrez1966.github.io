<b>Precondiciones:</b><br/>- Usuario identificado<br/>- El usuario es administrador general o administrador de la organización<br/><br/><b>Escenario principal:</b><br/>- Un usuario accede a la opción "Agregar usuario" de la lista de usuarios de la organización.<br/>- El sistema solicita el UID o DNI del usuario a agregar<br/> - Si el usuario ya está agregado a la organización, el sistema informa de ello al usuario.<br/>- El sistema localiza al usuario en DICODEF y muestra sus datos: uid, empleo, nombre y apellidos, y unidad de destino.<br/>- El usuario confirma el alta en la organización<br/>- Si el usuario no existe en la base de datos, se da de alta como entidad Usuario, con unas preferencias por defecto.<br/>- El sistema le asigna un rol de USUARIO asociado a la organización .<br/>- El sistema muestra la vista de detalle del usuario agregado, por si el administrador quiere modificar algún dato o agregarlo a algún grupo.<br/><br/><b>Postcondiciones:</b><br/>- El sistema ha enviado un mensaje de correo al usuario informándole del alta en el Gestor de Tareas de la organización.<br/><br/>
