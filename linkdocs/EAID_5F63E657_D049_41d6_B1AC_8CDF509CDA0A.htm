Buscador de usuario en DICODEF
 Permite buscar los datos de un usuario de DICODEF.
El botón "Comprobar" solo está habilitado si el campo "UID" no está vacío.

Muestra parte de los datos devueltos por la llamada al servicio de búsqueda de usuarios de DICODEF.

Habilitado solo si se ha localizado un usuario DICODEF que no exista ya en la aplicación (ver descripción funcional), y el bloque "Datos adicionales" tiene los datos obligatorios completados.

PROCESOS Y ELEMENTOS FUNCIONALES

 
TIPO
NOMBRE
 DESCRIPCION DEL PROCESO A EJECUTAR
Clic
Comprobar
 Realiza 2 acciones:
  • Primero comprueba si el usuario ya existe en la aplicación, intentando recuperar los datos a través del servicio REST de la entidad Usuario. Si el usuario ya existe en la aplicación, se informa de ello y se muestran en los campos los datos del usuario asociado.
  • Si el usuario no existe en la aplicación, llama al servicio API LeerUsuarioDicodef con el parámetro del UID. Si el servicio se ejecuta correctamente, se muestran los datos en el bloque "Datos de DICODEF". Si el servicio responde con un error, se muestra al usuario.
Clic
Selector de UCO (datos adicionales)
Muestra la vista "FormSeleccionUCO - Seleccionar UCO", con los parámetros por defecto, de forma que se muestre toda la Estructura Orgánica y se pueda seleccionar cualquier tipo de nodo.
Clic
Dar acceso al usuario
 Guarda el nuevo Usuario creado, con los datos especificados, llamando al servicio REST de la entidad Usuario.
Si la llamada es correcta se cierra el formulario y se devuelve como datos el nuevo usuario creado.
