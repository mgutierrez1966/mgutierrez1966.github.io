Opciones del bloque "Informes de recepción"
Toolbar
  • Emitir informe: Habilitado solo si:
    • El perfil es SEAPO, MANDO ORGANICO, CISPOC o GESTOR ALMACEN
    • Para el rol MANDO ORGANICO,  el ordenante de la reasignación es el Mando asignado al mismo
    • Para el rol CISPOC, la UCO apoyada por el mismo se corresponde con el ordenante de la reasignación, o bien es la UCO de destino de los recursos.

VISTA DIFERENCIADA POR ROLES
SI
SEAPO
Visibilidad de reasignaciones: Ve todas las reasignaciones
MANDO ORGANICO
 Ve las reasignaciones:
  • todas las ordenadas por Mando asignado al usuario entre UCO,s. subordinadas.
  • aquellas que afecten como origen o destino a UCO,s. de su ámbito en la estructura orgánica del ET.
CISPOC
 Ve las reasignaciones:
  • todas las ordenadas por la UCO apoyado por el usuario entre dependencias subordinadas.
  • aquellas que afecten como origen o destino a dependencias de su ámbito en la estructura orgánica del ET.
 
GESTOR ALMACEN
Ve las reasignaciones que tengan como origen o destino el almacén gestionado por el usuario.

Llama al servicio API EjecutarReasignacionHw. Si la llamada a la API es exitosa, se recarga la vista completa para actualizar la información y el estado de los componentes.

Clic
Borrar
 Solicita confirmación para eliminar permanentemente la reasignación del sistema.
Elimina el modelo.
Clic
Añadir recursos por consulta
 Muestra la vista "FormSeleccionRecursosHw", indicando como filtro previo la UCO de origen de la reasignacion.
Si la vista tiene exito, llama a la API AgregarRecursosReasignacionHw, pasando la lista de recursos seleccionados.
Si la llamada a la API tiene exito, se recarga el contenido del almacén asociado al grid de recursos reasignados, para reflejar los cambios.
 
Clic
Añadir recursos por lista de NI/NS
 ** INICIO ARI-015
Muestra la vista "FormSeleccionarRecursosHwPorLista".
Si la vista tiene éxito, se obtiene la información de los recursos mediante el servicio REST de RecursoHw, filtrando por los NI y NS especificados.
Si el servicio REST no devuelve ningún resultado, se informa de ello al usuario y se aborta la operación.
Si el servicio REST devuelve al menos un recurso, se extrae la lista de ID de los recursos, y se llama a la API AgregarRecursosReasignacionHw, pasando la lista de identificadores de recursos.
Si la llamada a la API tiene exito, se recarga el contenido del almacén asociado al grid de recursos reasignados, para reflejar los cambios.
**FIN ARI-015
