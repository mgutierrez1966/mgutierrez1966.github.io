- Para rol CISPOC: Se inicializan con los datos equivalentes de la UCO apoyada por el usuario.
- Para rol DIRECTOR: Los datos se inicializan en blanco

 DESCRIPCION DEL PROCESO A EJECUTAR
Inicializar vista
Para creación:
** INICIO ARIGIS-451
- Se llama a la API MantenimientoHwAPI.obtenerContratoMantenimientoHwEnVigor(), para obtener el objeto ContratoMantenimiento en vigor actualmente. Si la llamada devuelve un objeto nulo, se muestra un error al usuario informando de que "No se puede crear la solicitud porque no existe ningún contrato de mantenimiento de HW en vigor actualmente".
** FIN ARIGIS-451
- Crear un objeto SolicitudMantenimientoPerfectivo nuevo e inicializar valores por defecto:
      - criticidad: BAJA

Para edición:
- Tomar valores del objeto a editar
- Iniciar edición del modelo
- Recuperar la lista de recursos asociados a la solicitud.
Clic
Añadir recursos por consulta
 Muestra la vista "FormSeleccionRecursosHw", con los atributos:
- admiteSeleccionMultiple: true
- filtroPorEstado: ALTA

Si la vista tiene éxito, se añaden los recursos seleccionados a la lista de los recursos de la solicitud de mantenimiento, con las siguientes restricciones:
- Todos los recursos de la lista deben de pertenecer a la misma categoría hardware. En este caso se mostrará un error al usuario y no se añadirán los recursos.
- Un mismo recurso no puede estar aparecer más de una vez en la lista. En este caso no se mostrará error, pero los recursos duplicados no se añadirán.
 
Clic
Añadir recursos por lista de NI/NS
 Muestra la vista "FormSeleccionarRecursosHwPorLista".
Si la vista tiene éxito, se obtiene la información de los recursos mediante el servicio REST de RecursoHw, filtrando por los NI y NS especificados.
Si el servicio REST no devuelve ningún resultado, se informa de ello al usuario y se aborta la operación.
Si el servicio REST devuelve al menos un recurso, se añaden los recursos seleccionados a la lista de los recursos de la solicitud de mantenimiento, con las mismas restricciones que las indicadas para "Añadir recursos por consulta"
 
Clic
Quitar recursos
 Solicita confirmación para quitar de la reasignación los recursos seleccionados.
Elimina de la lista los recursos seleccionados.
 
Clic
Guardar
 Para edición:
- Finalizar edición del modelo
- Guardar el modelo (servicio REST de SolicitudMantenimientoPerfectivo )

Para creación:
- Si la lista de recursos asociados está vacía, se muestra el error "Especifique al menos un recurso a incluir en la solicitud de mantenimiento", y se cancela la operación.
- Llamar a la API MantenimientoHwAPI.CrearSolicitudPerfectivo(), pasando el nuevo objeto SolicitudMantenimientoPerfectivo y la lista de los identificadores de los recursos asociados a la nueva solicitud.
- Si la operación es correcta, se muestra un mensaje al usuario informando de que la solicitud se ha creado correctamente con el número XXXXXXX (devuelto por el servicio API).

En ambos casos, se guarda el objeto creado o modificado en la configuración de salida.
