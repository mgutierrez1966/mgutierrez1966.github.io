PROCESO
- Si el usuario no existe en el sistema (entidad Usuario):
        A - Se da de alta al usuario en la entidad. Los atributos "defXXXX" se inicializan con los datos del usuario en DICODEF, obtenidos del servicio GestionDicodefService.consultarDatosUsuarioDicodef de la librería ATLAS.
        B - Se autoriza al usuario para el acceso al sistema en DICODEF, mediante el servicio GestionDicodefService.autorizarUsuarioDicodef de la libreria ATLAS.
           C - Se crea una entidad PreferenciasUsuario asociada al nuevo usuario, con los valores por defecto.

- Se crea un rol USUARIO asociado al nuevo usuario, con el atributo "organizacion_GTAR" igual al ID de la organización.
- Se crea una entidad "Miembro" asociada al nuevo usuario. Se inicializa:
      - Nombre:  Empleo abreviado + Nombre y apellidos
      - Abreviatura: Empleo abreviado + Apellidos

- Se genera una notificación de tipo ALTA_USUARIO_ORGANIZACION para el nuevo usuario, de tipo "EMAIL".

PROCESO
- Se elimina, para el usuario pasado como parametro, el rol USUARIO que tenga como valor del atributo "organizacion_GTAR" el ID de la organizacion pasada como parametro. Si no existe ningún rol con esas características, se aborta la operación (sin generar error).
- Se elimina la entidad Miembro asociada al usuario.

- Se genera una notificación de tipo BAJA_USUARIO_ORGANIZACION para el usuario, de tipo "EMAIL".
