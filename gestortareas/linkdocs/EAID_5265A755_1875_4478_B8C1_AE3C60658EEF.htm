OBSERVACIONES
Titulo de la tarea
Etiqueta / Textfield
 Muestra el titulo de la tarea. Al pulsar en el icono de edición, se transforma en un TextField.

El icono de edición está solo visible si el usuario tiene el rol PROPIETARIO o RESPONSABLE_SUPERIOR sobre la tarjeta (API TableroAPI/rolesUsuarioEnTarjeta)
 
Etiquetas
Label
 Muestra una label por cada etiqueta asociada a la tarjeta.

El icono de edición está solo visible si el usuario tiene el rol PROPIETARIO o RESPONSABLE_SUPERIOR sobre la tarjeta.
 
En la lista...
Label
 Muestra la lista y el tablero donde se ubica la tarjeta.

Solo es visible si la tarjeta se encuentra asociada a una lista.
 
Subtarea de
Label
 Muestra el título de la tarea superior.

Si el usuario tiene el rol PROPIETARIO o RESPONSABLE_SUPERIOR sobre la tarjeta, se mostrará en modo enlace.

El bloque se muestra solamente si existe una tarea superior.
 
Fechas
Label
 Muestra las fechas en modo solo lectura. Las fechas de vencimiento y límite de actuación se muestran con el mismo código de color que en las minitarjetas de la vista PnTablero.
La fecha de vencimiento tiene un checkbox a su izquierda para marcarla como cumplida. Se asocia al atributo "finalizada" de Tarjeta.  El checkbox solo está habilitado si el usuario tiene el rol PROPIETARIO, RESPONSABLE o RESPONSABLE_SUPERIOR de la tarea. En caso de estar marcado la fecha se mostrará en texto blanco sobre fondo verde (ver boceto).

Los iconos para cambiar las fechas está solo visible si el usuario tiene el rol PROPIETARIO o RESPONSABLE_SUPERIOR sobre la tarjeta.

Al pulsar sobre el icono de calendario, se mostrará el selector de fecha para seleccionar la fecha correspondiente. Se guardarán los datos con la operación UPDATE del REST de Tarjeta
 
Descripción
Text / Textarea
 Muestra o edita la descripción de la tarea. El icono de edición está solo visible si el usuario tiene el rol PROPIETARIO o RESPONSABLE_SUPERIOR sobre la tarjeta.
 
Subtareas
Tab Panel
 Muestra entre paréntesis el número de subtareas de la tarjeta.

En la cabecera:
- Muestra el botón "Añadir" para añadir una nueva subtarea. Visible solo si el usuario tiene el rol RESPONSABLE sobre la tarjeta.
- Una barra de progreso con el porcentaje de subtareas finalizadas (atributo "finalizada" a TRUE). Solo visible si existen subtareas. Si el botón "Añadir" no está visible, ocupará todo el ancho de la pestaña.
 
Bloque de subtarea
Se repite para cada subtarea de la tarjeta.

Se compone de:

- Título de la subtarea. Se muestra solo si la subtarea tiene un responsable asignado. Si el usuario tiene el rol RESPONSABLE sobre la tarjeta o sobre la subtarea, se mostrará en modo enlace.
- Checkbox  a su izquierda para marcarla como cumplida. Se asocia al atributo "finalizada" de la subtarea.Si está marcada, el texto de la subtarea se mostrará en gris y tachado (ver boceto)
- Responsable: mostrará el icono de persona o de grupo, en función del tipo de participante asignado. También muestra la abreviatura del responsable.
- Fechas: Muestra las fechas de inicio y de vencimiento, de la misma forma que para la tarea, salvo que no se muestra el checkbox de "finalizada", ya que está en el título.
- Botón "Eliminar": Se muestra el usuario tiene el rol RESPONSABLE sobre la tarjeta.
 
Comentarios
Tab Panel
 Muestra entre paréntesis el número de comentarios de la tarjeta (en su espacio de trabajo).

La lista de comentarios se muestra ordenada por fecha de forma ascendente (más antiguo primero).


En la cabecera muestra:
- Botón "Añadir" para añadir un comentario. Visible solo si el usuario tiene el rol PROPIETARIO, RESPONSABLE, AUTORIZADO, COLABORADOR O RESPONSABLE_SUBTAREA sobre la tarjeta.

- Botón "Eliminar" para eliminar un comentario. Visible solo si el usuario tiene el rol PROPIETARIO o RESPONSABLE sobre la tarjeta, o si el usuario es el autor del comentario.
 
Documentos
Tab Panel
 Muestra entre paréntesis el número de documentos adjuntos de la tarjeta (en su espacio de trabajo).

En la cabecera muestra:
- Botón "Añadir" para añadir un documento. Visible solo si el usuario tiene el rol PROPIETARIO, RESPONSABLE, AUTORIZADO, COLABORADOR O RESPONSABLE_SUBTAREA sobre la tarjeta.

- Botón "Eliminar" para eliminar un documento. Visible solo si el usuario tiene el rol PROPIETARIO o RESPONSABLE sobre la tarjeta, o si el usuario es el autor del comentario.

En la columna "Nombre" se muestra el nombre del documento, en forma de enlace.
 
Actividad
Tab Panel
 Muestra la lista de actividades asociadas a la tarjeta.

PROCESOS Y ELEMENTOS FUNCIONALES

 
TIPO
NOMBRE
 DESCRIPCION DEL PROCESO A EJECUTAR
Iniciar vista
PENDIENTE DE DETERMINAR DE DONDE OBTENER LOS DATOS

Inicializa los datos del formulario, y la visibilidad de los botones y controles.
 
Clic
Edición de titulo
 Muestra un textfield para editar el título. Al pulsar Intro o hacer clic fuera del textfield, se guarda el dato (REST de Tarjeta) y se pasa a modo solo lectura.
 
Clic
Edición de etiquetas
 Muestra la vista FormSeleccionEtiquetas, pasando la lista de etiquetas asignadas a la tarjeta.

Al recuperar el control, si se ha realizado algún cambio, llama a la API TableroAPI/establecerEtiquetasTarjeta, pasando el ID de la tarjeta, la lista completa de etiquetas seleccionadas, y el parámetro "agregar" a FALSE. La API devuelve la lista actualizada de las etiquetas asignadas a la tarjeta.
 
Clic
Titulo de tarea superior
 Si el modo enlace está activado, se muestra una nueva vista de FormTarjeta, pasando la tarea a visualizar.
 
Clic
Selector de fecha (inicio, vencimiento, limite)
 Muestra el selector de fecha inicializado con el valor a editar.

Al recuperar el control, actualiza la fecha mostrada en la vista ( operación UPDATE del REST Tarjeta)

 
Clic
Checkbox de fecha de vencimiento
 Se actualiza el atributo "finalizada" de la tarea con el nuevo valor del checkbox (true o false), y se llama a la operación UPDATE del REST de Tarjeta, para guardar la tarea.

Se actualiza la visualización (colores) de la fecha de vencimiento,según esté o no finalizada.
 
Clic
Edición de descripción
 Muestra un editor de textArea para editar la descripción.

El icono de edición se transforma en un icono de "Guardar" (ver boceto).
 
Clic
Icono "Guardar" de "Descripción"
 Guarda el valor de la descripción (operacion UPDATE del REST de Tarjeta) y pasa a modo solo lectura. El icono se transforma de nuevo en "Editar".
 
Clic
Subtareas - Añadir
 Muestra la vista FormCrearTarea.

Al recuperar el control, actualiza la lista de subtareas para mostrar la nueva subtarea.
 
Clic
Titulo de subtarea
 Si el modo enlace está activado, se muestra una nueva vista de FormTarjeta, pasando la subtarea a visualizar.
 
Clic
Checkbox de subtarea
 Se actualiza el atributo "finalizada" de la subtarea con el nuevo valor del checkbox (true o false), y se llama a la operación UPDATE del REST de Tarjeta, para guardar la subtarea.

Se actualiza el porcentaje completado de la Tarjeta, y que se visualiza en la barra de progreso.

Se actualiza la visualización del titulo de la subtarea (mostrar como enlace o tachado), en función de si está o no finalizada.

Se actualiza la visualización (colores) de la fecha de vencimiento,según esté o no finalizada.
 
Clic
Subtarea - Eliminar
 Se solicita confirmación al usuario, indicando que la operación de borrado es permanente.

Se llama la operación DELETE del REST de Tarjeta para eliminar la subtarea.
 
Clic
Comentarios - Añadir
 Se muestra la vista FormComentario para añadir un nuevo comentario, pasando el ID de la tarjeta.

Al recuperar el control se actualiza la lista de comentarios.
 
Clic
Comentarios - Eliminar
 Se solicita confirmación al usuario.

Se elimina el comentario llamando a la operacion DELETE del REST de Comentario.

Al recuperar el control, se actualiza la lista de comentarios.
 
Clic
Documentos - Añadir
 Muestra el explorador de ficheros del sistema para seleccionar un documento.
USAR COMPONENTE GESTION DE DOCUMENTOS DE FRONT.
 
Clic
Documentos - Nombre de documento
 El fichero asociado al documento se abre en ventana nueva (si es tipo PDF), o se descarga (resto de tipos).
 
Clic
Documentos - Eliminar
 Se solicita confirmación al usuario.

Se elimina el comentario llamando a la operacion DELETE del REST de Adjunto.

Al recuperar el control, se actualiza la lista de documentos.
 
Clic
Asignar a...
 Se muestra la vista FormAsignarResponsable, pasando el id de la tarjeta.
Al recuperar el control, se actualiza en la vista los datos del responsable con los datos del REST de Tarjeta.
 
Clic
Transferir tarea
 Se muestra la vista FormTransferirTarea pasando el id de la tarjeta.
Al recuperar el control, se actualiza en la vista los datos del responsable con los datos del REST de Tarjeta.
 
Clic
Colaboradores
 Se muestra la vista FormColaboradores, pasando el id de la tarjeta.

Al recuperar el control, se actualiza el número de colaboradores que se muestra entre paréntesis al lado del nombre del responsable.
 
Clic
Enviar a un tablero
 Se muestra la vista FormEnviarTarjetaTablero, pasando el ID de la tarjeta.

Al recuperar el control, se actualiza la vista con los datos de la lista y el tablero asignados a la tarjeta.
 
Clic
Guardar como plantilla
 Muestra la vista FormGuardarPlantillaTarjeta, pasando el identificador de la tarjeta.
 
Clic
Archivar tarea
 Se solicita confirmación de la operación. Informa al usuario de que la tarjeta desaparecerá de su tablero y de la lista de tareas, pero se podrá recuperar posteriormente por el propietario del tablero o por el administrador de la organización.

Llama a la API TableroAPI/archivarTarjeta, pasando el identificador de la tarjeta.

Al recuperar el control, se cierra la vista.
 
Clic
Eliminar tarea
 Se solicita confirmación antes de su eliminación, indicando que la eliminación será permanente, y se borrarán también las subtareas asociadas. Informa también de que existe la opción "Archivar tarjeta" para dejar de visualizar la tarjeta y poder recuperarla posteriormente.

Se llama la operación DELETE del REST de Tarjeta para eliminar la tarea.

Al recuperar el control, se cierra la vista.
