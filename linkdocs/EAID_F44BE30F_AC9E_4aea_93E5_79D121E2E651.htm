Panel de detalle de una solicitud de mantenimiento perfectivo para recursos Hw. Además de la consulta de datos, permite lanzar operaciones de tramitación de la solicitud.
CONFIGURACION ENTRADA
  • solicitudMantenimientoPerfectivo: Requerido. Solicitud a mostrar.

ELEMENTOS VISUALES

 
OBSERVACIONES
Barra de opciones
Opciones disponibles:
- Modificar solicitud: Activo en alguno de estos casos:
      - Si la solicitud está en estado "Nueva", y el rol es CISPOC
** INICIO ARIGIS-1136
      - Si el rol es DIRECTOR, en cualquier estado de la solicitud.
** FIN ARIGIS-1136

- Borrar solicitud: elimina la solicitud. Activo solo si la solicitud está en estado "Nueva", y el rol es CISPOC.

** INICIO ARIGIS-1108
- Crear parte de queja: Activo solo si la solicitud está en estado "Aprobada por Director", "Comunicada a empresa" o "Resuelta por la empresa", y el rol es CISPOC.
** FIN ARIGIS-1108

- Tramitar solicitud: Despliega un menu con las opciones de tramitación disponibles. Varían según rol y estado:
ROL CISPOC: El menú no se muestra para este rol.
ROL OACISE: El menú se muestra si el estado es "Nueva". Opciones:
  - Denegar solicitud
  - Tramitar a Subdirector
ROL SUBDIRECTOR: El menú se muestra si el estado es "Tramitada a Subdirector por OACISE". Opciones:
  -  Denegar solicitud
  - Tramitar a Director
ROL DIRECTOR: El menú se muestra si el estado es "Tramitada a Director por Subdirector". Opciones:
  - Aprobar solicitud
  - Denegar solicitud
** INICIO ARIGIS-848
ROL RT22: El menú no se muestra para este rol
** FIN ARIGIS-848
 
Datos de la solicitud
Panel con datos de la solicitud y del recurso HW, organizados en grupos de campos.
** INICIO ARIGIS-451
El grupo "Contrato de mantenimiento" muestra los datos del objeto ContratoMantenimiento de la solicitud.
** FIN ARIGIS-451
El panel ajusta su altura al contenido a mostrar, con un máximo de 2/3 del contenedor (permite scroll vertical).
Pestaña "Recursos asociados"
Dispone de una barra de opciones y de la lista de recursos asociados a la solicitud. En el título de la pestaña muestra el número de recursos asociados.
Barra de opciones de "Recursos asociados"
Toolbar
 Opciones de la lista de recursos:
- Añadir recursos por consulta: Habilitado en alguno de los siguientes casos:
      * El rol es CISPOC y el estado de la solicitud es NUEVA
      * El rol es DIRECTOR y el estado de la solicitud es TRAMITADA A DIRECTOR POR SUBDIRECTOR.
- Añadir recursos por lista de NI/NS: Habilitado en los mismos casos que la opción anterior.
- Quitar recursos: Habilitado si hay seleccionado algún recurso en la lista, y además se cumple alguno de los casos anteriores.
- Ver recurso: Habilitado si hay seleccionado algún recurso en la lista.
Tabla de recursos asociados
Grid
 Grid con la lista de recursos asociados a la solicitud de mantenimiento perfectivo.
Toma los datos del servicio REST de la entidad SolicitudMantPerfecHwRec, aplicando el filtro con la solicitud de mantenimiento.
Inicialmente ordenada por NI ascendente.
Pestaña "Tramitación"
Lista ordenada por defecto por fecha ascendente. Pestaña activa por defecto.
El grid obtiene los valores del servicio REST de la entidad SolicitudMantenimientoTramit
Pestaña "Partes de queja"
Lista ordenada por defecto por fecha ascendente. El grid obtiene los valores del servicio REST de la entidad ParteQueja.
Cada fila del grid puede desplegar un panel que contiene otro grid con el histórico de tramitación del parte de queja. Este grid obtiene los datos del servicio REST de la entidad ParteQuejaTramitacion.
 
Pestaña "Resolución"
Visualiza los datos del informe de resolución de la empresa, organizados en grupos. Obtiene la información de la solicitud actual.
 
Pestaña "Partes de trabajo"
Lista ordenada por defecto por nombre de documento ascendente. El grid obtiene los valores del servicio REST de la entidad SolMantoDoc.
La columna "Documento" contiene un enlace al servicio de descarga de ficheros, con los parámetros para descargar el fichero asociado al documento.

PROCESOS Y ELEMENTOS FUNCIONALES

 
TIPO
NOMBRE
 DESCRIPCION DEL PROCESO A EJECUTAR
Carga inicial
Se recuperan datos de la solicitud de mantenimiento pasada como parametro. Entidad: SolicitudMantenimientoPerfectivo.
Se recuperan datos de los recursos asociados a la solicitud (si no vienen incluidos en la misma). Entidad SolicitudMantPerfecHwRec.
Se recuperan las tramitaciones realizadas en la solicitud. Entidad SolicitudMantenimientoTramit.
Botón pulsado
Modificar solicitud
 Se muestra la vista FormSolicitudMtoPerfecHW para editar la solicitud activa.
Si la vista se realiza correctamente, se actualiza la información mostrada de la solicitud de mantenimiento.
Botón pulsado
Borrar solicitud
 Se solicita confirmación al usuario.
Se llama al servicio REST de la entidad SolicitudMantenimientoPerfectivo para realizar el borrado.
Se cierra la vista.
Botón pulsado
Crear parte de queja
 Se muestra la vista FormParteQueja pasándole la solicitud de mantenimiento actual.
Si la vista finaliza correctamente, se actualiza la pestaña "Partes de queja".
 
Botón pulsado
Tramitar solicitud
Despliega las opciones disponibles para la tramitación (ver apartado "Elementos")
Menú seleccionado
Submenus de "Tramitar solicitud"
 En función del tipo de trámite solicitado:

- Denegar solicitud: Se muestra la vista FormMotivosDenegacionSolManto, para recoger los motivos de la denegación.
-  Tramitar a XXXXX: Se muestra la vista FormObservacionesTramitacionSolManto, para recoger las observaciones a la tramitación de la solicitud.
- Aprobar solicitud: Se muestra la vista FormObservacionesAprobacionSolManto, para recoger las observaciones a la tramitación de la solicitud.

Si las vistas anteriores son correctas, se llama al servicio API MantenimientoHwApi.tramitarSolicitudManto, pasando la acción solicitada, las observaciones y la solicitud de mantenimiento.

Si la respuesta es correcta, se actualiza la información completa de la vista.
Clic
Añadir recursos por consulta
 Muestra la vista "FormSeleccionRecursosHw", con los atributos:
- admiteSeleccionMultiple: true
- filtroPorEstado: ALTA

Si la vista tiene éxito, se llama a la API MantenimientoHwApi.agregarRecursosPerfectivo(), pasando la lista de recursos seleccionados.
Si la llamada a la API tiene exito, se recarga el contenido del almacén asociado al grid de recursos asociados, para reflejar los cambios.
 
Clic
Añadir recursos por lista de NI/NS
 Muestra la vista "FormSeleccionarRecursosHwPorLista".
Si la vista tiene éxito, se obtiene la información de los recursos mediante el servicio REST de RecursoHw, filtrando por los NI y NS especificados.
Si el servicio REST no devuelve ningún resultado, se informa de ello al usuario y se aborta la operación.
Si el servicio REST devuelve al menos un recurso, se llama a la API MantenimientoHwApi.agregarRecursosPerfectivo(), pasando la lista de recursos seleccionados.
Si la llamada a la API tiene exito, se recarga el contenido del almacén asociado al grid de recursos asociados, para reflejar los cambios.
 
Clic
Quitar recursos
 Solicita confirmación para quitar de la reasignación los recursos seleccionados.
Se llama a la API MantenimientoHwApi.quitarRecursosPerfectivo(), pasando la lista de recursos seleccionados.
Si la llamada a la API tiene exito, se recarga el contenido del almacén asociado al grid de recursos asociados, para reflejar los cambios.
 
Clic
Ver recurso
Abre en una nueva pestaña la vista PnRecursoHWDetalle con los datos del recurso seleccionado. La nueva pestaña puede cerrarse.
Enlace pulsado
 Enlace del documento "parte de trabajo"
 (pestaña "Partes de trabajo")
Llama al servicio API LeerFicheroDocumento, pasando el identificador del documento asociado al parte de trabajo seleccionado. El servicio devuelve un documento PDF para descargar o visualizar en pantalla.
Clic en simbolo > del parte de queja
Pestaña Partes de queja
 Muestra u oculta la tabla de detalle de las tramitaciones del parte de queja.
